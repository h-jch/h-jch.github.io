定时器的基础是ScheduledThreadPoolExecutor

ScheduledThreadPoolExecutor#

ScheduledThreadPoolExecutor类支持周期性任务的调度

继承ThreadPoolExecutor#

  1. 任务封装成ScheduledFutureTask对象，继承FutureTask，基于相对时间，不受系统时间的改变所影响
  2. 使用DelayedWorkQueue延时队列，有序队列，通过每个任务距离下次执行时间的间隔来排序

public ScheduledFuture<?> schedule(Runnable command,
                                   long delay,
                                   TimeUnit unit) {
    if (command == null || unit == null)
        throw new NullPointerException();
    RunnableScheduledFuture<?> t = decorateTask(command,
        new ScheduledFutureTask<Void>(command, null, triggerTime(delay, unit)));
    delayedExecute(t);
    return t;
}

public <V> ScheduledFuture<V> schedule(Callable<V> callable,
                                       long delay,
                                       TimeUnit unit) {
    if (callable == null || unit == null)
        throw new NullPointerException();
    RunnableScheduledFuture<V> t = decorateTask(callable, 
				new ScheduledFutureTask<V>(callable, triggerTime(delay, unit)));
    delayedExecute(t);
    return t;
}

支持Runnable对象和Callable对象，将传入的任务封装为RunnableScheduledFuture对象（就是ScheduledFutureTask对象）

通过delayedExecute方法延时执行任务

public ScheduledFuture<?> scheduleAtFixedRate(Runnable command,
                                              long initialDelay,
                                              long period,
                                              TimeUnit unit) {
    if (command == null || unit == null)
        throw new NullPointerException();
    if (period <= 0)
        throw new IllegalArgumentException();
    ScheduledFutureTask<Void> sft =
        new ScheduledFutureTask<Void>(command,
                                      null,
                                      triggerTime(initialDelay, unit),
                                      unit.toNanos(period));
    RunnableScheduledFuture<Void> t = decorateTask(command, sft);
    sft.outerTask = t;
    delayedExecute(t);
    return t;
}

如果任务执行的时间超过了period，下一次任务会被立即执行

public ScheduledFuture<?> scheduleWithFixedDelay(Runnable command,
                                                 long initialDelay,
                                                 long delay,
                                                 TimeUnit unit) {
    if (command == null || unit == null)
        throw new NullPointerException();
    if (delay <= 0)
        throw new IllegalArgumentException();
    ScheduledFutureTask<Void> sft =
        new ScheduledFutureTask<Void>(command,
                                      null,
                                      triggerTime(initialDelay, unit),
                                      unit.toNanos(-delay)); // 负数表示相对固定的延迟执行
    RunnableScheduledFuture<Void> t = decorateTask(command, sft);
    sft.outerTask = t;
    delayedExecute(t);
    return t;
}

private long triggerTime(long delay, TimeUnit unit) {
    return triggerTime(unit.toNanos((delay < 0) ? 0 : delay));
}

long triggerTime(long delay) {
    return now() +
        ((delay < (Long.MAX_VALUE >> 1)) ? delay : overflowFree(delay));
				// 判断是否要防止Long类型溢出
}

防止Long类型溢出方法：overflowFree方法，该方法限制延迟队列中所有节点的延迟时间在Long.MAX_VALUE之内，防止在执行compareTo方法中溢出

private void delayedExecute(RunnableScheduledFuture<?> task) {
    // 如果线程池已经关闭，使用拒绝策略拒绝任务
    if (isShutdown())
        reject(task);
    else {
        // 添加到阻塞队列中
        super.getQueue().add(task);
        if (isShutdown() &&
            !canRunInCurrentRunState(task.isPeriodic()) &&
            remove(task))
            task.cancel(false);
        else
            // 确保线程池中至少有一个线程启动，即使corePoolSize为0
            // 该方法在ThreadPoolExecutor中实现
            ensurePrestart();
    }
}

void ensurePrestart() {
    int wc = workerCountOf(ctl.get());
    if (wc < corePoolSize)
        addWorker(null, true);
    else if (wc == 0)
        addWorker(null, false);
				// 线程池中的工作线程通过addWorker方法来启动并执行任务
}

线程池中的工作线程通过addWorker方法来启动并执行任务，工作线程会不断地从阻塞队列中取出任务并执行（调用任务的run方法）

public void run() {
    // 是否是周期性任务
    boolean periodic = isPeriodic();
    // 当前线程池运行状态下如果不可以执行任务，取消该任务
    if (!canRunInCurrentRunState(periodic))
        cancel(false);
    // 如果不是周期性任务，调用FutureTask中的run方法执行
    else if (!periodic)
        ScheduledFutureTask.super.run();
    // 如果是周期性任务，调用FutureTask中的runAndReset方法执行
    // runAndReset方法不会设置执行结果，所以可以重复执行任务
    else if (ScheduledFutureTask.super.runAndReset()) {
        // 计算下次执行该任务的时间
        setNextRunTime();
        // 重复执行任务
        reExecutePeriodic(outerTask);
    }
}

outerTask就是task本身，在scheduledAtFixedRate方法和scheduledAtFixedRate方法中设置了sft.outerTask = t;

void reExecutePeriodic(RunnableScheduledFuture<?> task) {
    if (canRunInCurrentRunState(true)) {
        super.getQueue().add(task);
        if (!canRunInCurrentRunState(true) && remove(task))
            task.cancel(false);
        else
            ensurePrestart();
    }
}

调用reExecutePeriodic方法时此周期任务已经被执行过一次了

DelayedWorkQueue#

基于最小堆结构的优先队列，保证子节点的下次执行时间比父节点的下次执行时间要大，定时任务执行的时候需要取出最近要执行的任务

向DelayedWorkQueue中添加任务的时候优先队列都会进行调整，以保证最小堆的特性

public RunnableScheduledFuture<?> take() throws InterruptedException {
    final ReentrantLock lock = this.lock;
    lock.lockInterruptibly();
    try {
        for (;;) {
            RunnableScheduledFuture<?> first = queue[0];
            if (first == null)
                available.await();
            else {
                // 计算当前时间到执行时间的时间间隔
                long delay = first.getDelay(NANOSECONDS);
                if (delay <= 0)
                    return finishPoll(first);
                first = null; // don't retain ref while waiting
                // leader不为空，阻塞线程
                if (leader != null)
                    available.await();
                else {
                    // leader为空，则把leader设置为当前线程，
                    Thread thisThread = Thread.currentThread();
                    leader = thisThread;
                    try {
                        // 阻塞到执行时间
                        available.awaitNanos(delay);
                    } finally {
                        // 设置leader = null，让其他线程执行available.awaitNanos(delay);
                        if (leader == thisThread)
                            leader = null;
                    }
                }
            }
        }
    } finally {
        // 如果leader不为空，则说明leader的线程正在执行available.awaitNanos(delay);
        // 如果queue[0] == null，说明队列为空
        if (leader == null && queue[0] != null)
            available.signal();
        lock.unlock();
    }
}

private RunnableScheduledFuture<?> finishPoll(RunnableScheduledFuture<?> f) {
    // 数组长度-1
    int s = --size;
    // 取出最后一个节点
    RunnableScheduledFuture<?> x = queue[s];
    queue[s] = null;
    // 长度不为0，则从第一个元素开始排序，目的是要把最后一个节点放到合适的位置上
    if (s != 0)
        siftDown(0, x);  // 调整堆
    setIndex(f, -1);
    return f;
}

take方法一般在getTask方法中调用，工作线程会循环地从workQueue中取任务，对于定时任务来讲，要保证只有在到指定的执行时间任务才能被取走

Scheduled定时器#

工作流程#

  1. 通过注解@EnableScheduling中的@Import引入SchedulingConfiguration，配置了一个ScheduledAnnotationBeanPostProcessor
  2. ScheduledAnnotationBeanPostProcessor解析和处理每一个Bean中的@Scheduled注解，将解析完成的方法封装为不同类型的Task，缓存在ScheduledTaskRegistrar中
  3. ScheduledAnnotationBeanPostProcessor中的钩子接口方法afterSingletonsInstantiated()和onApplicationEvent()方法（这两个方法互斥，只有一个方法能被调用），在所有单例初始化完成后回调触发，设置任务调度器（TaskScheduler或者ScheduledExecutorService，默认实现类是ScheduledThreadPoolExecutor），将所有Task实例添加到任务调度器中执行

定时任务Task#

  1. cron表达式任务，CronTask
  2. 固定延迟间隔任务，FixedDelayTask
  3. 固定频率任务，FixedRateTask

任务的解析和封装都在ScheduledAnnotationBeanPostProcessor这个类中的processScheduled()方法中
